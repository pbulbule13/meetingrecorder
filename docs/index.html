Nexus Meeting Recorder

AI-Powered Meeting Intelligence Platform - Local-First, Privacy-Focused

Nexus Meeting Recorder is a production-grade AI meeting intelligence platform that captures, processes, and transforms conversations into actionable knowledge. Built with Electron and Python microservices, it provides real-time transcription, AI-powered insights, and comprehensive knowledge management while keeping all your data local and secure.

Task-Based Routing

The LLM service intelligently routes requests to the optimal provider based on task type:
  • Intent Detection: EURI ‚Üí Groq ‚Üí Gemini ‚Üí GPT-4o
  • Code Generation: EURI ‚Üí GPT-4o ‚Üí Groq ‚Üí Gemini
  • Summarization: EURI ‚Üí Gemini ‚Üí Groq ‚Üí GPT-4o ‚Üí Claude
  • General: EURI ‚Üí DeepSeek ‚Üí Gemini ‚Üí GPT-4o ‚Üí Groq

Advanced Features

  • Token Tracking: Real-time cost estimation across all providers
  • Prompt Optimization: Automatic token reduction while preserving quality
  • Semantic Caching: 1-hour TTL for identical requests
  • Fallback Strategy: Automatic failover for 99.9% uptime

Installation

Quick Start (Windows)

# Clone repository
git clone https://github.com/pbulbule13/meetingrecorder.git
cd meetingrecorder

# Run automated setup
start_services.bat

Manual Setup

# 1. Install Node.js dependencies
npm install

# 2. Create Python virtual environment
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. Copy environment template
copy .env.example .env  # Windows
cp .env.example .env    # Linux/Mac

# 5. Edit .env with your API keys
# 6. Start services
start_services.bat  # Windows
Important: You need at least one API key for STT and one for LLM to use the full functionality. Free tiers available with Groq and EURI.

API Endpoints

Health Checks

GET /health

Available on all services. Returns service status and configuration.

Transcription Service (Port 38421)

POST /transcribe/stream

Transcribe an audio chunk in real-time. Returns text with timestamps.

POST /transcribe/file

Transcribe a complete audio file. Supports WAV, MP3, MP4.

LLM Service (Port 45231)

POST /complete

General text completion with automatic provider fallback.

POST /summarize

Generate comprehensive meeting summary with key topics.

POST /extract/action-items

Extract actionable tasks with assignees and due dates.

POST /extract/decisions

Extract key decisions made during the meeting.

GET /stats

Get token usage and cost statistics across all providers.

RAG Service (Port 53847)

POST /query

Semantic search across all indexed meetings.

POST /index/meeting

Index a new meeting in the knowledge base.

POST /prepare-meeting

Generate pre-meeting briefing with historical context.

DELETE /meeting/{meeting_id}

Remove a meeting from the knowledge base index.

Troubleshooting

Services Won't Start

Problem: Python services fail to start or port is already in use. 
# Check if ports are in use (Windows)
netstat -ano | findstr ":38421 :45231 :53847"

# Kill existing Python processes
taskkill /F /IM python.exe

No Audio Captured

Windows: Enable "Stereo Mix" in Sound Settings
macOS: Grant microphone permissions in System Preferences
Linux: Ensure PulseAudio is running

Transcription Returns "All Providers Failed"

  • Verify API keys in .env file
  • Ensure at least one STT provider is configured
  • Check API provider status and rate limits

LLM Service Slow Response

# Check token usage stats
curl http://127.0.0.1:45231/stats

# Reset stats if needed
curl -X POST http://127.0.0.1:45231/stats/reset

RAG/Knowledge Base Not Working

  • Check ChromaDB health: GET http://127.0.0.1:53847/health
  • Verify chromadb and sentence-transformers are installed
  • Note: RAG is optional; system works without it
