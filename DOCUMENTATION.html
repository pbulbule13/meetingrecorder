üìñ Usage Guide

1. Recording a Meeting

  1. Start Recording: Click the "Start Recording" button in the main window
  2. Name Your Meeting: Provide a title or let it auto-generate
  3. View Live Transcript: Watch the real-time transcription appear
  4. Use Real-Time Assistance: Open the floating overlay (Ctrl+Shift+O) for instant help
  5. Stop Recording: Click "Stop" when finished
  6. Review Results: Summary, action items, and analytics are generated automatically

2. Meeting Preparation

  1. Connect Calendar: Go to Settings ‚Üí Integrations ‚Üí Connect Google Calendar/Outlook
  2. View Upcoming Meetings: Navigate to "Upcoming" tab in dashboard
  3. Prepare for Meeting: Click "Prepare" button on any upcoming meeting
  4. Review Insights:
    • Relevant past discussions
    • Key topics to cover
    • Suggested talking points
    • Anticipated questions
    • Recommended agenda

3. Searching Knowledge Base

# Full-text search Search: "database migration" # Semantic search (natural language) Search: "What did we decide about the API design?" # Filter by date Search: "budget" + Date Range: Last 30 days # Filter by participant Search: "action items" + Assigned to: John Doe

4. Using the Local API

# Generate API key (from Settings or API) curl -X POST http://localhost:62194/api/v1/auth/keys \ -H "Authorization: Bearer <existing-key>" \ -d '{"name": "My Integration"}' # List all meetings curl http://localhost:62194/api/v1/meetings \ -H "Authorization: Bearer nxs_your_api_key" # Search transcripts curl "http://localhost:62194/api/v1/search?q=database&limit=20" \ -H "Authorization: Bearer nxs_your_api_key" # Get action items curl http://localhost:62194/api/v1/action-items?status=pending \ -H "Authorization: Bearer nxs_your_api_key" # Export meeting curl -X POST http://localhost:62194/api/v1/meetings/abc-123/export \ -H "Authorization: Bearer nxs_your_api_key" \ -H "Content-Type: application/json" \ -d '{"format": "md"}' > meeting.md

üîß Troubleshooting

Common Issues

Issue: Audio capture not working

Solutions:
  • Verify FFmpeg is installed: ffmpeg -version
  • On Windows: Enable "Stereo Mix" in Sound Settings
  • Grant microphone permissions to the app
  • Check audio device selection in Settings

Issue: Service won't start (port conflict)

Solutions:
  • Check if another app is using the port: netstat -ano | findstr :PORT
  • Change the port in .env file
  • Restart the application

Issue: LLM requests failing

Solutions:
  • Verify API keys are correct in .env
  • Check API provider status pages
  • Review rate limits on your account
  • Check logs: ~/nexus-assistant/logs/app.log

Logs Location

  • Application: ~/nexus-assistant/logs/app.log
  • Python Services: ~/nexus-assistant/logs/python.log
  • Database: ~/nexus-assistant/data/nexus.db
